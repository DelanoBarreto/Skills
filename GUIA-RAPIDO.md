## 🎓 Dicas Para Iniciante

### 1. **Comece pelo `/ajuda`**
   Quando tiver dúvida sobre qual skill usar, digite `/ajuda` e descreva o que quer fazer.

### 2. **Combine Skills em Sequência**
   ```
   /brainstorming    → /planejamento    → /writing-plans    → /frontend (ou /backend)
   ```
   Primeira: clarear ideia. Depois: planejar. Depois: implementar.

### 3. **Erros Comuns**
   - ❌ Tentar codificar sem `/brainstorming` → output confuso
   - ❌ Testar sem `/testes` definir estratégia → testes inúteis
   - ❌ Commitar sem `/requesting-code-review` → bugs na produção
   - ✅ Sempre: planejamento → implementação → teste → review → commit

### 4. **Não Precisa Ativar Todas**
   Use as skills mais relevantes para seu projeto. Se for backend puro, `/frontend` não é necessário.

### 5. **Skills Estão Aqui Para Ajudar**
   Uma skill NÃO faz o trabalho por você. Uma skill **ensina o Claude** como ajudá-lo melhor.

## ❓ FAQ

**P: Por que não vejo todas as skills quando digito `/` ?**  
R: Tem 43+ skills instaladas, mas o menu `/` só mostra as que estão no nível raiz. Use `/ajuda` para acessar todas.

**P: Qual skill uso para X?**  
R: Digite `/ajuda` e descreva o que quer fazer. A skill vai sugerir as melhores opções.

**P: Posso usar 2 skills ao mesmo tempo?**  
R: Sim! Ativa uma, trabalha, depois ativa outra. Skills funcionam em sequência.

**P: Meu projeto é governamental, por onde começo?**  
R: `/portalgov-system` (se for PortalGovMunicipal), depois `/planejamento`, depois `/backend-api` ou `/frontend`.

**P: Qual é a diferença entre `/testes` e `/playwright` ?**  
R: `/testes` = planejamento de testes. `/playwright` = implementação de testes E2E específicos.
