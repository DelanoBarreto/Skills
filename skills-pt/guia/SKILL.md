# 🎯 Guia de Skills e Agentes — O Que Usar em Cada Situação

Use este guia quando **não souber por onde começar** ou **qual skill/agente escolher** para sua tarefa.

---

## 🚨 Estou com um ERRO ou BUG

**Comportamento estranho, erro na build, falha em teste ou regressão?**

→ **`/debug`** — investigar qualquer erro, crash, comportamento inesperado  
→ **`@debugger`** — quando o bug é complexo e precisa de análise profunda

---

## 🎨 Quero criar ou alterar a TELA / INTERFACE

**Componente, página, botão, formulário ou layout?**

→ **`/frontend`** — criar/alterar componentes React, páginas, formulários  
→ **`/design`** — dúvida é visual: layout, cores, acessibilidade  
→ **`@frontend-specialist`** — problema complexo de arquitetura de UI

---

## 🔌 Quero criar ou alterar a API / SERVIDOR

**Rota, serviço, validação, integração com API externa?**

→ **`/backend`** — criar rota, serviço, validação, integração  
→ **`@backend-specialist`** — quando envolve segurança, arquitetura complexa

---

## 🗄️ Quero trabalhar com BANCO DE DADOS

**Tabela, query, migration, índice ou schema?**

→ **`/banco-de-dados`** — criar tabela, query, migration, índice  
→ **`/supabase`** — específico Supabase (Auth, RLS, policies)  
→ **`@database-architect`** — modelagem complexa

---

## ✅ Quero escrever ou revisar TESTES

**Testes unitários, integração, E2E ou cobertura?**

→ **`/testes`** — escrever testes unitários, integração, TDD  
→ **`/qa-automacao`** — automatizar testes no navegador (E2E com Playwright)  
→ **`@test-engineer`** — estratégia de testes e cobertura

---

## 👀 Quero revisar o CÓDIGO antes de fazer merge

**Revisar PR, checar qualidade, procurar bugs?**

→ **`/revisao`** — revisar PR, checar qualidade, encontrar bugs

---

## 🔐 Tenho dúvida de SEGURANÇA

**Riscos, autenticação, dados sensíveis ou vulnerabilidades?**

→ **`/seguranca`** — verificar riscos, auth, dados sensíveis, boas práticas  
→ **`@security-auditor`** — auditoria profunda e defensiva

---

## ⚡ Quero melhorar a PERFORMANCE

**App lento, queries lentas ou bundle grande?**

→ **`/performance`** — analise e otimização de performance  
→ **`@performance-optimizer`** — análise profunda de métricas

---

## 🔄 Quero configurar CI/CD ou GITHUB ACTIONS

**Pipeline falhando, workflow ou PR checks?**

→ **`/github-ci`** — checks, Actions, falhas de pipeline

---

## 📋 Preciso PLANEJAR uma feature ou projeto

**Decompor trabalho, definir escopo, priorizar etapas?**

→ **`/planejamento`** — planejar e decompor trabalho  
→ **`@project-planner`** — planejar projetos inteiros

---

## 📖 Quero criar DOCUMENTAÇÃO

**README, guia, changelog ou especificação?**

→ **`/documentacao`** — criar/revisar documentação técnica  
→ **`@documentation-writer`** — documentação completa e profissional

---

## 🤔 Não sei por onde começar / Tarefa COMPLEXA

**Tarefa envolve múltiplas áreas ou não sabe qual skill usar?**

→ **`/roteador`** — descreva o que quer fazer e recebe a skill certa  
→ **`@orchestrator`** — coordena múltiplos agentes em paralelo

## 🎯 Quando Usar Skill vs Agente?

| Situação | Use |
|---|---|
| **Tarefa simples e bem definida** | **Skill** (`/nome`) — resposta rápida, PT-BR |
| **Tarefa complexa ou com múltiplas partes** | **Agente** (`@nome`) — análise profunda |
| **Não tem certeza** | Comece com a **skill**, escale para o **agente** se precisar |

---

## 💡 Dicas Práticas

1. **Sempre comece aqui** → Se não souber qual skill usar, este `/guia` te ajuda
2. **Combine ferramentas** → Use skill para prototipagem rápida, depois agente para análise profunda
3. **Não tenha medo de testar** → Teste `/frontend` ou `/debug` e ajuste conforme o resultado
4. **Para tarefas novas** → Use `/roteador` ou descreva o que quer fazer
