# PortalGov System

Apply this skill whenever work touches the PortalGov product, architecture, UI, transparency modules, admin area, or tenant data model.

## Product Rules

- Treat PortalGov as a SaaS GovTech for municipal portals.
- Preserve native multi-tenancy via `src/app/[tenant]` and tenant-aware links.
- Never hardcode a tenant slug into shared components or reusable data flows.
- Prefer data-driven sections configured by `TenantConfig` now and Supabase later.
- Public portal content must prioritize transparency, accessibility, performance, and trust.

## Current UI Rules

- Use the active radius system:
  - Home page: `8px` / `rounded-md`.
  - Internal pages, dashboards, dense data views: `6px` / `rounded-sm` or `rounded`.
  - Micro-elements, tags, compact badges: `4px` / `rounded-xs` when available.
- Keep the look technical, dense, clean, and institutional.
- Use Lucide React icons. Do not use emojis as interface icons.
- Use `max-w-[1440px]`, `mx-auto`, and the existing layout components.
- Keep Home sections dense, generally `pt-12 pb-24`.
- Use zebra section backgrounds (`bg-white`, `bg-slate-50`) for readability.
- Use `InternalPageShell` for internal pages.
- Keep breadcrumbs generated by the shell or shared breadcrumb system, not manually duplicated.
- Use "Gestao Municipal" in institutional navigation instead of isolated "Prefeito" labels.

## Transparency And PNTP

For transparency modules, ensure:

- Public access without login.
- Visible "Atualizado em" date.
- At least 3 years of historical data when applicable.
- Export actions for CSV, JSON, and/or PDF where the data type supports it.
- Granular filters such as year, period, status, category, keyword, or department.
- WCAG AA contrast and keyboard-accessible controls.

## Supabase, Admin, And Security

- Model data around `municipio_id` or equivalent tenant key.
- Use Supabase Auth for users and roles.
- Plan roles at minimum: `superadmin`, `admin`, `editor`.
- Use Row Level Security for tenant isolation.
- Use soft delete for CMS-managed content unless there is a legal reason for physical deletion.
- Keep public-read data separate from authenticated admin write paths.
- Do not expose service-role keys to the browser.

## Admin Media Rules

- Prefer WebP or modern JPG for uploaded images.
- Require alt text in the admin for accessibility and SEO.
- Keep images compressed; use `250KB` as the target maximum unless a section explicitly needs more.
- Hero images should support wide crops, preferably around `1920x800`.
- Avoid text baked into images when the UI renders title/description overlays.
